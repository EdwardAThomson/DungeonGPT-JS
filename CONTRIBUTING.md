# Contributing to DungeonGPT-JS

Thanks for your interest in contributing.

## Ground Rules

- All contributions to this repository are made under the Apache License 2.0.
- Inbound equals outbound: by submitting code, you agree your contribution is licensed under Apache-2.0.
- Do not submit code you do not have the legal right to contribute.

## Required for Pull Requests

- Use a branch and open a pull request.
- Keep commits focused and clearly described.
- Sign off each commit with DCO certification.
- Sign your commits (GPG or SSH signing) so authorship is verifiable.

### DCO Sign-off

Use one of the following approaches:

- Add `-s` when committing:
  - `git commit -s -m "your message"`
- Or add this trailer manually to each commit message:
  - `Signed-off-by: Your Name <your-email@example.com>`

By signing off, you certify the Developer Certificate of Origin (DCO):
https://developercertificate.org/

## Review Expectations

- Pull requests should include enough context to explain the change and rationale.
- Changes may be requested for security, maintainability, licensing, or product direction.
- Maintainers may decline contributions that create legal or provenance risk.

## Security and Secrets

- Never commit secrets, API tokens, private keys, or credentials.
- If your change affects authentication or API handling, document security implications in the PR.

## Questions

If you are unsure whether a contribution is acceptable, open an issue first and describe the proposal.
